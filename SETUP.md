# RockUP Token Audit — Инструкция по установке

Пайплайн: **Figma Plugin → Apps Script Web App → Google Sheets**

---

## Структура файлов

```
figma-plugin/
  manifest.json   — описание плагина
  code.js         — main thread (читает переменные Figma)
  ui.html         — интерфейс плагина

apps-script/
  Code.gs         — Google Apps Script (создаёт/обновляет Sheets)
```

---

## Часть 1 — Google Apps Script Web App

### 1.1 Создать проект

1. Открой [script.google.com](https://script.google.com) → **Новый проект**
2. Назови его `RockUP Token Audit`
3. Удали весь дефолтный код
4. Вставь содержимое файла `apps-script/Code.gs`
5. Сохрани (`Ctrl+S` / `Cmd+S`)

### 1.2 Задеплоить как Web App

1. **Развернуть** → **Управление развёртываниями** → **Новое развёртывание**
2. Тип: **Веб-приложение**
3. Параметры:
   - **Выполнять как:** Я (`diceygimmik91@gmail.com`)
   - **У кого есть доступ:** Все (анонимно) — иначе Figma не сможет POST
4. Нажми **Развернуть** → скопируй URL вида:
   ```
   https://script.google.com/macros/s/AKfycb…/exec
   ```

> **Важно:** при каждом изменении кода нужно создавать новое развёртывание —
> старый URL продолжает работать со старой версией.

## Часть 2 — Figma Plugin

### 2.1 Установить плагин локально (для разработки)

1. Открой Figma Desktop
2. Меню → **Plugins** → **Development** → **Import plugin from manifest...**
3. Укажи путь к `figma-plugin/manifest.json`
4. Плагин появится в разделе **Development plugins**

### 2.2 Запустить аудит

1. Открой файл `⭐ Rock UP – UI Kit` в Figma
2. `Ctrl+Alt+P` / `Cmd+Option+P` → **Plugins** → **Development** → **RockUP Token Audit**
3. Вставь URL из шага 1.2
4. Нажми **Запустить аудит** ▶

При первом запуске Apps Script создаёт новую Google Таблицу автоматически. ID таблицы сохраняется в Script Properties — последующие запуски обновляют ту же таблицу.

Чтобы зафиксировать таблицу вручную: скопируй ID из URL листа (часть между `/d/` и `/edit`) и вставь в поле **Spreadsheet ID** в плагине.

---

## Часть 3 — Структура Google Sheets

После первого запуска таблица будет содержать 7 листов:

| Лист | Содержание |
|------|-----------|
| **Summary** | Метрики аудита + топ shared токенов |
| **Matrix** | Матрица токен × компонент |
| **Token overlaps** | Токены, используемые в 2+ компонентах |
| **Component details** | Полная таблица: cmp-токен → sys-токен → ref |
| **Light Dark** | Маппинг Light/Dark alias по всем токенам |
| **WCAG 2.1 AA** | Контрастность FG/BG пар (авто из ref-цветов) |
| **Alias duplicates** | Токены с одинаковыми ref-значениями |

## Обновление

Когда в дизайн-системе что-то меняется — просто открой плагин и нажми **Запустить аудит** снова. Данные в таблице обновятся.

---

## Решение проблем

**Плагин не видит коллекции `NEW Sys / Theme`**
→ Убедись, что файл RockUP UI Kit открыт и библиотека подключена.

**Ошибка сети при отправке**
→ Проверь, что Apps Script задеплоен с доступом **Все (анонимно)**, и URL скопирован полностью.

**Данные отправляются, но таблица не обновляется**
→ Проверь **Apps Script → Executions** — там будет текст ошибки, если что-то пошло не так на стороне скрипта.

**WCAG-лист пустой**
→ Плагин не нашёл FG/BG пары в одних и тех же секциях компонентов. Это нормально для компонентов, где цвета заданы без явных fg/bg токенов.
