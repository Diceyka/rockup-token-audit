# RockUP Token Audit — Инструкция по установке

Пайплайн: **Figma Plugin → Apps Script Web App → Google Sheets**

---

## Структура файлов

```
figma-plugin/
  manifest.json                             — описание плагина
  code.js                                   — main thread (читает переменные Figma)
  ui.html                                   — интерфейс плагина
  Code.gs_v6_refcolors_safe_diagnostics.gs  — зеркало живого Apps Script backend'а (см. ниже)
```

> Раньше был отдельный каталог `apps-script/` с `Code.gs` — он удалён (коммит
> `97b56c1: remove apps-script`). Backend теперь живёт **только в самом
> Google Apps Script проекте** ("RockUP Audit 2026" на script.google.com),
> а `figma-plugin/Code.gs_v6_refcolors_safe_diagnostics.gs` в репозитории —
> это ручное зеркало его исходников для ревью/истории. Редактирование этого
> файла в репо **не** обновляет живой скрипт — изменения нужно вручную
> скопировать в редактор Apps Script (см. 1.2).

---

## Часть 1 — Google Apps Script Web App

### 1.1 Обновить существующий проект

Проект уже создан и задеплоен — с нуля создавать новый не нужно.

1. Открой [script.google.com](https://script.google.com) → проект `RockUP Audit 2026`
2. Внеси изменения в `Код.gs` (тот же код, что в
   `figma-plugin/Code.gs_v6_refcolors_safe_diagnostics.gs` — держи их в синхроне вручную)
3. Сохрани (`Ctrl+S` / `Cmd+S`)

### 1.2 Обновить деплой (не создавать новый!)

1. **Начать развёртывание** → **Управление развёртываниями**
2. Найди активное развёртывание (иконка карандаша) → **Изменить**
3. В поле **Версия** выбери **Новая версия**, впиши краткое описание изменения
4. **Начать развёртывание**

> **Важно:** используй именно "Новая версия" существующего развёртывания —
> так `/exec` URL остаётся прежним. Если вместо этого создать **новое**
> развёртывание, оно получит **новый** URL, а старый останется указывать на
> старую версию кода. Именно так репозиторий/плагин когда-то накопили 15+
> развёртываний с разными URL, и `code.js` начал ссылаться на устаревший.
> URL, который сейчас захардкожен в `figma-plugin/code.js`
> (`TEAM_DEFAULTS.scriptUrl`), должен совпадать с URL активного
> развёртывания — сверяй после каждого обновления.

## Часть 2 — Figma Plugin

### 2.1 Установить плагин локально (для разработки)

1. Открой Figma Desktop
2. Меню → **Plugins** → **Development** → **Import plugin from manifest...**
3. Укажи путь к `figma-plugin/manifest.json`
4. Плагин появится в разделе **Development plugins**

### 2.2 Запустить аудит

1. Открой файл `⭐ Rock UP – UI Kit` в Figma
2. `Ctrl+Alt+P` / `Cmd+Option+P` → **Plugins** → **Development** → **RockUP Token Audit**
3. Вставь URL из шага 1.2
4. Нажми **Запустить аудит** ▶

При первом запуске Apps Script создаёт новую Google Таблицу автоматически. ID таблицы сохраняется в Script Properties — последующие запуски обновляют ту же таблицу.

Чтобы зафиксировать таблицу вручную: скопируй ID из URL листа (часть между `/d/` и `/edit`) и вставь в поле **Spreadsheet ID** в плагине.

## Часть 3 — Структура Google Sheets

После первого запуска таблица будет содержать 9 листов (порядок задан `SHEET_ORDER`
в Code.gs):

| Лист | Содержание |
|------|-----------|
| **Журнал изменений** | История изменений токенов между аудитами |
| **Dashboard** | Сводные метрики + счётчики Action items по критичности |
| **Matrix** | Матрица токен × компонент |
| **Light Dark** | Маппинг Light/Dark alias по всем токенам |
| **Component details** | Полная таблица: cmp-токен → sys-токен → ref |
| **WCAG** | Контрастность FG/BG пар (Pass/Fail, авто из ref-цветов) |
| **Token overlaps** | Токены, используемые в 2+ компонентах |
| **Ref Colors** | Все ref-цвета с hex и визуальным свотчем |
| **Action items** | Auto-generated из WCAG/alias-дублей/shared-токенов; smart-merge — ручные статусы и владельцы сохраняются между запусками |

> Alias-дубликаты и "Summary"-метрики больше не отдельные листы — они попадают
> в **Action items** (тип "Alias duplicate") и **Dashboard** соответственно.

## Обновление

Когда в дизайн-системе что-то меняется — просто открой плагин и нажми **Запустить аудит** снова. Данные в таблице обновятся.

---

## Решение проблем

**Плагин не видит коллекции `NEW Sys / Theme`**
→ Убедись, что файл RockUP UI Kit открыт и библиотека подключена.

**Ошибка сети при отправке**
→ Проверь, что Apps Script задеплоен с доступом **Все (анонимно)**, и URL скопирован полностью.

**Данные отправляются, но таблица не обновляется**
→ Проверь **Apps Script → Executions** — там будет текст ошибки, если что-то пошло не так на стороне скрипта.

**WCAG-лист пустой**
→ Плагин не нашёл FG/BG пары в одних и тех же секциях компонентов. Это нормально для компонентов, где цвета заданы без явных fg/bg токенов.
